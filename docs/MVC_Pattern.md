# MVC Pattern

## Overview

The Customer Registry application follows the MVC (Model-View-Controller) architecture.

### Model

The Model represents application data and database schemas.

Examples:
- User Model
- Customer Model
- Complaint Model

### View

The View represents the user interface built using React.js.

Examples:
- Login Page
- Dashboard
- Customer Page
- Complaint Page

### Controller

Controllers handle requests, process business logic, and communicate with models.

Examples:
- Authentication Controller
- Customer Controller
- Complaint Controller

## MVC Workflow

User
↓
View (React.js)
↓
Routes
↓
Controller
↓
Model
↓
MongoDB

## Benefits

- Better Code Organization
- Easy Maintenance
- Scalability
- Reusability
- Separation of Concerns
